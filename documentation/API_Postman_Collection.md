# API Integration Specs (Postman Collection)

**Project:** Enterprise Event Management & ROI Tracking Portal  
**Company:** PROPERTY SHOP INVESTMENT LLC  
**Purpose:** To define the exact request and response payloads required for the CRM to push read-only data into the Event Portal's Firebase database via secure webhooks.

> **Security Mandate:** All endpoints require a valid JWT Bearer Token. Requests missing the `Authorization: Bearer <token>` header will be strictly rejected with a 401 Unauthorized status.

---

## 🧑💼 Endpoint 1: Staff & Hierarchy Sync
**Purpose:** Upserts (creates or updates) an agent or manager profile to ensure the portal routing always uses the latest company hierarchy.

*   **HTTP Method:** `POST`
*   **Endpoint Path:** `/api/v1/sync/users`
*   **Headers:**
    *   `Content-Type: application/json`
    *   `Authorization: Bearer <Your_JWT_Token>`

### Request Body Schema:
*   `crm_user_id` (String, Required): The absolute source-of-truth ID. Example: `"USR-84729"`
*   `full_name` (String, Required): Example: `"Amr ElFangary"`
*   `email` (String, Required): Example: `"propertyshopinvest@gmail.com"`
*   `company_entity` (String, Required): Example: `"PROPERTY SHOP INVESTMENT LLC"`
*   `branch_name` (String, Required): Example: `"Abu Dhabi HQ"`
*   `department` (String, Required): Example: `"Off-Plan Sales"`
*   `nationality` (String, Optional): Example: `"UAE"`
*   `languages` (Array of Strings, Optional): Example: `["English", "Arabic"]`
*   `manager_id` (String, Required): The CRM ID of this person's direct approver. Example: `"USR-11223"`

### Success Response (200 OK):
*   `status` (String): `"success"`
*   `message` (String): `"User profile upserted successfully."`
*   `timestamp` (String): ISO-8601 formatted date string.

## 🏢 Endpoint 2: PSI Maps Property Inventory Sync
**Purpose:** Pushes active real estate developments into the portal so organizers can tag events with specific inventory, triggering the correct L&D assessments.

*   **HTTP Method:** `POST`
*   **Endpoint Path:** `/api/v1/sync/projects`
*   **Headers:**
    *   `Content-Type: application/json`
    *   `Authorization: Bearer <Your_JWT_Token>`

### Request Body Schema:
*   `project_id` (String, Required): The unique ID from the PSI Maps database. Example: `"PRJ-551"`
*   `developer_name` (String, Required): Example: `"Aldar Properties"`
*   `project_name` (String, Required): Example: `"Mamsha Gardens"`
*   `project_tier` (String, Required): Must be exactly one of: `"Luxury"`, `"Medium"`, or `"Average"`. Example: `"Luxury"`
*   `status` (String, Required): Example: `"Active"`

### Success Response (200 OK):
*   `status` (String): `"success"`
*   `message` (String): `"Project inventory synced successfully."`
*   `processed_id` (String): Matches the sent `project_id`.

### Error Response (400 Bad Request - Validation Error):
*   `status` (String): `"error"`
*   `message` (String): `"Invalid project_tier. Must be Luxury, Medium, or Average."`

## 💰 Endpoint 3: Post-Event Revenue Sync (Actuals)
**Purpose:** Feeds finalized, closed deals back into the portal months after the roadshow to close the loop on the 50/30/20 commission ROI dashboard.

*   **HTTP Method:** `PATCH`
*   **Endpoint Path:** `/api/v1/events/{event_id}/actuals`
*   **Headers:**
    *   `Content-Type: application/json`
    *   `Authorization: Bearer <Your_JWT_Token>`
    *   `Idempotency-Key` (String, Required): A unique hash generated by the CRM for this specific transaction to prevent double-counting if a network timeout occurs.

### Request Body Schema:
*   `agent_id` (String, Required): The specific agent who closed the deals. Example: `"USR-84729"`
*   `deals_closed` (Number, Required): Total count of finalized transactions. Example: `2`
*   `actual_gross_revenue` (Number, Required): The final AED amount used for the commission splits. Example: `850000`
*   `transaction_date` (String, Required): ISO-8601 formatted date string.

### Success Response (200 OK):
*   `status` (String): `"success"`
*   `message` (String): `"Event actuals updated. Dashboard recalculation triggered."`
*   `event_id` (String): Matches the path parameter.

### Error Response (409 Conflict - Idempotency):
*   `status` (String): `"error"`
*   `message` (String): `"Transaction already processed. Idempotency key match."`
